Information terminology:
Cookie：
It is a mechanism that allows the server (or script) to store and retrieve information on the client, and it adds a simple and continuous client end state to a Web-based client/server application. The server sends a state message to the client when returning the HTTP object, which is saved on the client. The valid URL range in this state is explained in the information of state. Thereafter, all HTTP requests initiated by the client within the URL will return the current value of the state information (the so-called Cookie) from the client to the server.
SSID： Service Set Identifier is used to distinguish different networks. Different networks can be accessed after entering corresponding passwords for different SSIDs set in the wireless network cards. In brief, mutual communication can be achieved only when the SSIDs, as the name of a local area network, are set to the same value.
MAC address: Media Access Control, refers to the physical address of the network interface or the so-called hardware address. In OSI model and TCP/IP protocol stack, MAC addresses are used for respective forwarding of data packet in the corresponding data link layer and network interface layer.
